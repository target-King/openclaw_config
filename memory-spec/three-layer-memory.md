# 三层记忆设计

## 目标
在不把整段历史反复塞进上下文的前提下，保留真正有价值的信息。

## Dialog 隔离原则

当系统支持多对话框并发时，记忆层需要按 `dialogId` 隔离：

| 记忆层 | 绑定 dialogId | 绑定 topic | 说明 |
|---|---|---|---|
| L1 | 是 | 是 | 每个对话框的实时上下文完全独立 |
| L2 | 是 | 是 | 同 topic 不同对话框的摘要不混写 |
| L3 | 否 | 否 | 全局共享的稳定知识，所有对话框可读 |

- L1/L2 的读写操作必须携带 `dialogId`，避免跨对话框污染
- L3 不绑定 dialogId——它保存的是跨对话框共享的长期事实
- 如果多个对话框在同一 topic 下得出相似 L2 结论，可由 supervisor 精炼后提升到 L3

## L1：当前会话记忆
用途：
- 保存最近几轮上下文
- 优先回答当前问题

特点：
- 时效短
- 读频高
- 不长期保存
- 按 dialogId + topic 隔离存储

## L2：话题摘要记忆
用途：
- 把某个 topic 的长对话压缩为摘要
- 在后续同 topic 检索时快速回忆

特点：
- 中期保存
- 面向主题
- 摘要优先
- 按 dialogId + topic 隔离存储

## L3：长期事实记忆
用途：
- 保存稳定规则、命名规范、路径约定、用户长期偏好

特点：
- 写入门槛高
- 保存时间长
- 读取价值高
- 全局共享，不绑定 dialogId

## 默认检索顺序
L1 -> L2 -> L3

检索 L1/L2 时必须按当前 dialogId 过滤。检索 L3 时不过滤 dialogId。

## 默认写入规则
- 临时噪音：不写入
- 会反复用到的主题结论：写入 L2（绑定当前 dialogId）
- 稳定规则和偏好：写入 L3（不绑定 dialogId）
