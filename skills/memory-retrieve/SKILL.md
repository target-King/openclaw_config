# memory-retrieve

## 目的
根据当前问题，从三层记忆里取出最相关的上下文，而不是把整段历史全部塞回上下文。

## 适用场景
- 需要回忆某个主题此前讨论过什么
- 当前问题依赖历史规则、路径、偏好
- 对话过长，不能直接携带全部历史

## 默认检索顺序
1. L1：当前会话最近片段
2. L2：话题摘要
3. L3：长期事实

## 输入
- 当前问题
- dialogId（**必须**，用于 L1/L2 的 dialog 隔离过滤）
- topic（可选）
- 关键词（可选）
- topK（可选）

## 输出
- 精简后的上下文片段
- 话题摘要
- 长期事实候选
- 命中的来源层级

## 执行原则
- 能从 L1 解决，就不继续下探
- 需要规则时，优先读 L2 / L3
- 返回摘要，不返回整库
- 不返回敏感信息
- L1/L2 检索时按 dialogId 过滤，不跨 dialog 取数据
- L3 检索不过滤 dialogId，全局共享

## 配套脚本
- `scripts/memory/retrieve_context.py --dialog-id <id> --topic <topic>`
