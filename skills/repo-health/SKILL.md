# repo-health

## 目的
检查控制仓库结构是否完整，避免后面同步时才发现缺文件。

## 检查内容
- 必需目录是否存在
- 每个 agent 是否有 4 个核心文件
- 每个 skill 是否有 `SKILL.md`
- `managed-config` 是否齐全
- 记忆脚本是否存在
- `.env.example` 与 `.gitignore` 是否存在

## 输出
- 通过项
- 缺失项
- 警告项
- 建议修复项

## 配套脚本
- `scripts/doctor.ps1`

## 原则
- 尽量在本地仓库阶段发现问题
- 不等同步后才报错
- 输出要清楚指出具体路径
