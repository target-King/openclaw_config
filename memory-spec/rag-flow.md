# RAG / 检索流程

## 当前阶段
当前先做“轻量 RAG”：
- SQLite
- topic 路由
- 关键词检索
- 摘要检索
- 长期事实检索

## 输入流程
1. 收到问题
2. 先判断 topic
3. 检索 L1 最近上下文
4. 若不足，再检索 L2 摘要
5. 仍不足，再检索 L3 长期事实
6. 拼装精简上下文
7. 输出回答
8. 视情况回写 L2 / L3

## 设计原则
- 先 topic，后检索
- 先摘要，后原始片段
- 少量高质量上下文优先于大量原文堆砌
- 当前先不强依赖 embedding

## 后续增强
- embedding 检索
- hybrid search
- 多 topic 聚合
- relevance 打分
