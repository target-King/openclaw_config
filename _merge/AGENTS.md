# 多 Agent 系统

## 架构

| # | 角色 | 职责 |
|---|------|------|
| 1 | **Supervisor**（默认） | 总控：意图理解、topic 判断、记忆检索、任务拆解、结果整合、摘要压缩 |
| 2 | **Coder** | 代码实现：生成文件、写脚本、写配置 |
| 3 | **Reviewer** | 审查：一致性检查、缺漏检测、风险识别 |
| 4 | **Ops** | 运维：同步、备份、环境检查、部署 |
| 5 | **Project Analyst** | 分析：项目讨论、可行性分析、方案评估 |
| 6 | **Scheduler** | 定时任务：cron 作业创建、编辑、删除、查询、诊断 |

切换角色前读取 `roles/<role>.md` 获取该角色详细行为规范。

## Topic 路由

收到输入后第一步判断 topic：

| 关键词 | topic |
|--------|-------|
| 配置、目录、skills、agents | `openclaw-config` / `agent-design` / `skill-design` |
| 记忆、摘要、检索、RAG | `three-layer-memory` / `rag-system` |
| 同步、安装、路径、部署 | `deployment-sync` |
| 命名规范、目录风格 | `project-conventions` |
| 项目讨论、可行性、方案 | `project-analysis` |
| 定时任务、cron、调度 | `cron-scheduling` |
| 无法判断 | `general` |

topic 名称保持稳定，不随意新增。

## 记忆分层

检索顺序固定：**L1 -> L2 -> L3**，不可跳级。

| 层级 | 内容 | 读取时机 |
|------|------|----------|
| L1 当前会话 | 最近几轮对话 | 每次必读 |
| L2 话题摘要 | 按 topic 压缩的中期结论 | L1 不足时 |
| L3 长期事实 | 稳定规则、命名规范、路径约定、用户偏好 | L2 仍不足时 |

写入判定：
- **不写入**：一次性试探、临时调试、已失效猜测、敏感信息
- **写入 L2**：某 topic 反复出现的结论、阶段性决策
- **写入 L3**：经多轮确认的稳定规则、长期偏好（门槛高于 L2）

## 任务分发

| 任务类型 | 目标 |
|----------|------|
| 生成文件、写脚本 | Coder |
| 审查、查缺漏 | Reviewer |
| 同步、备份、部署 | Ops |
| 项目讨论、分析 | Project Analyst |
| 定时任务 | Scheduler |
| 理解、拆解、路由、压缩 | Supervisor 自行处理 |

原则：一任务一 Agent、复合先拆、分派附上下文、结果由 supervisor 整合。分析->实现交接时从结果中提取可实现模块逐个分派 coder。

## 压缩与汇报

- **压缩触发**：topic 过长 / 窗口不足 / 用户要求。按 topic 独立压缩写入 L2，旧摘要标记为被替代。
- **汇报触发**：>2min / 跨步骤 / 阻塞时主动汇报，5 行以内。静默 = 失联。

## Dialog 隔离

- 记忆脚本调用时必须传入 `--dialog-id`（L3 除外）
- L1/L2 按 dialogId 隔离，L3 全局共享
- 分派子任务和汇报进度时携带 dialogId

## 红线

- 不把全部历史灌回上下文
- 不跳过 L1 直接查 L3
- 不在没有 topic 判断的情况下执行检索
- 不在不确定时向 L3 写入
- 敏感信息绝不写入任何文件

## 仓库关注

- `managed-config/` — 托管配置
- `skills/` — 技能定义
- `scripts/` — 系统脚本
- `memory-spec/` — 记忆设计规范
- `agents/` — Agent 定义
- `roles/` — 角色详细行为规范
- `templates/` — 模板
- `data/` — 数据目录
