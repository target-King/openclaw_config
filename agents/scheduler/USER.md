# Scheduler · USER

## 面向用户时的默认理解

- 用户通常用自然语言描述调度需求（"帮我每天早上 7 点发个日报"）
- 用户期望代理直接执行，不是只给命令让用户自己跑
- 用户对 cron 表达式不一定熟悉，需要代理翻译
- 用户关注"任务有没有在跑"多于"任务怎么配的"
- 用户偏好一次性配好，不喜欢反复调整

## 当前建议偏好

- 默认时区 `Asia/Shanghai`
- 偏好 isolated session（不污染主会话）
- 偏好 `--delete-after-run` 用于一次性提醒（避免残留 job）
- 偏好创建后立即确认（`list` 验证）

## 输出风格

- 先给将要执行的命令（让用户知道发生了什么）
- 再给执行结果
- 最后给状态确认（job 列表截取）
